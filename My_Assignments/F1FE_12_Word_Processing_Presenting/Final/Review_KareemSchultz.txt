2.1.1 Software Justification — Why a Word Processor for the Welcome Pack
==========================================================================

When I first looked at this assignment, I genuinely considered just doing the whole Welcome Pack in PowerPoint since I was already going to be working with it for the Digital Signage presentation anyway. But after actually thinking about what the Welcome Pack needed to do, I realised pretty quickly that a word processor was the right call, and honestly the only sensible option for this kind of document.

The biggest reason comes down to how much text is involved. The Welcome Pack has multiple paragraphs of running prose — the introduction, the venue details, speaker bios, and so on. Word processors are fundamentally built for handling large blocks of text. You get proper paragraph formatting, control over line spacing, widow and orphan control, and the text just flows naturally from one page to the next. In PowerPoint, text lives inside individual text boxes on separate slides, which means you are constantly fighting the layout when the content is longer than a few bullet points. There is no concept of a multi-page flowing document in a presentation tool.

Then there is the Table of Contents. This was a huge factor. In Microsoft Word (or LibreOffice Writer), you mark your section headings with Heading 1 style, then go to References and insert an automated Table of Contents. The software literally generates it for you based on those styles, complete with page numbers that update automatically if the content shifts around. PowerPoint has absolutely nothing like this. You would have to manually type out a contents slide and update the slide numbers yourself every time anything changed. For a professional conference document, that is just not practical.

The mail merge functionality was another decisive factor. The assignment required creating personalised ID badges for all 10 delegates using data from the Delegate_List spreadsheet. Word processors have a built-in mail merge wizard that connects directly to an Excel data source, lets you insert merge fields like Title, FirstName, LastName, Company, and then generates individual outputs for each record. I used this to create a badge grid on Page 4 with each delegate's details automatically populated. Trying to replicate this in PowerPoint would mean either manually creating each badge by hand or writing some kind of VBA macro, which defeats the purpose.

Finally, the document needed a structured, print-ready layout with consistent headers, footers showing "Page X of Y", properly formatted tables with specific column widths, and an image with text wrapping. These are all core features of word processing software. PowerPoint is designed for screen projection, not for producing multi-page reference documents that people might print out and carry around at a conference.

So in short, the word processor gave me automated ToC, mail merge, proper pagination, text flow, and print-ready layout — none of which PowerPoint handles well, if at all.

2.1.2 Efficiency Review — Repetitive Task and Improvement Suggestion
==========================================================================

The most repetitive part of the Word Processing task was definitely formatting the delegate ID badges on Page 4. Each badge needed the same basic structure: the conference header text at the top, then the delegate's name (built from Title, FirstName, and LastName fields), their company underneath, and their category (Speaker, VIP, or Delegate) at the bottom. With 10 delegates, I had to set up this identical formatting structure for every single badge cell in the table, adjusting fonts, sizes, alignment, and colours each time.

What made it particularly tedious was the colour-coding for categories. Speakers were green, VIPs were red, and regular delegates were blue. I had to manually apply the right colour for each badge depending on the category, which meant checking each delegate's category and then changing the font colour accordingly.

If I were doing this again, or if there were even more delegates (imagine a conference with 200 people), I would create a Word template with a pre-formatted badge layout and use it alongside the mail merge feature more extensively. Specifically, I could set up a macro in VBA that automatically applies conditional formatting based on the Category field — something like an IF field that checks whether the category is "VIP" and applies red formatting, "Speaker" for green, and so on. Word actually supports IF merge fields natively (like { IF { MERGEFIELD Category } = "VIP" "VIP" "Delegate" }), which I could combine with character formatting to automate the whole colour-coding process. This would save significant time and reduce the chance of making mistakes on individual badges.

2.1.3 Audience Review — Digital Signage vs Speaker-Led Keynote
==========================================================================

The Digital Signage presentation I created was designed to run unattended on screens in the conference lobby, which fundamentally shaped every design decision I made. But if the same content needed to be presented as a Speaker-Led Keynote — say, during the opening ceremony with someone standing on stage talking through the slides — I would change quite a lot about how the presentation works.

First, I would remove all the interactive Action Buttons. The digital signage version has clickable navigation buttons (Agenda, Map, Wifi, Home) because attendees are meant to walk up and tap the screen themselves. In a keynote, the speaker controls the flow, so navigation is entirely linear — you just go from slide 1 to slide 2 to slide 3 in order. Interactive elements would be confusing and unnecessary.

Second, I would turn off the kiosk looping mode. The digital signage is set to loop continuously until someone presses Escape, because it needs to keep cycling through information all day. A keynote runs once, from start to finish, with the speaker manually advancing each slide. I would also remove the automatic timings and let transitions happen on click only.

Third, I would significantly reduce the text on each slide. The signage version has quite detailed agenda listings and Wi-Fi instructions because people need to read everything themselves. In a keynote, the speaker is the main source of information, and slides should just be visual support — maybe a few key words or an image, not paragraphs of text. I would move the detailed content into the Speaker Notes section instead, so the presenter can reference it without cluttering the slides.

Finally, I would add proper slide transitions and build animations to make the presentation more dynamic when someone is presenting live, rather than the static displays used for signage.
